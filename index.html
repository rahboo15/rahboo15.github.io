Twitter's Bootstrap

Simplified Front-end Development

We have all seen and used it: the messy world wide web. It is full of languages, styles, "standards," frameworks, and preferences. We both love and hate it for this mixture. On the one hand, it is very flexible and simple to create a website that is versatile. On the other, it can become overwhelming to maintain. Although it does not fix all issues, I know of a solution to make life simpler that both seasoned web developers and new comers to the industry can appreciate: Twitter's Bootstrap.

Brief History

Even popular Twitter suffered from the clutter of web design and development. In the beginning, developers used whatever library they were familar and comfortable with. Because of this, many inconsistencies within Twitter began to emerge. These inconsistencies cause issues with both maintanence and scalability. This called for a time of consolidation and standardization.

According to Twitter, Bootstrap began as an answer to these challenges and quickly accelerated during Twitter's Hackweek. By the end of Hackweek, we had reached a stable version that engineers could use accross the company.1 Thanks to the efforts of Twitter employees Mark Otto and Jacob Thornton, a framework for consistency was then available for all internal web tools.2

Since that time, engineers have improved upon the original Bootstrap. It is now open to the public and even depends on the public for future improvements. I personally am acquainted with Bootstrap 2.3.2; however, it is currently in its third version. Further on, I will explain some of the differences and why you should use Bootstrap version 3.x if possible.

Using Bootstrap vs Bootstrapping

When people hear Bootstrap, especially in the business world, they often think of bootstrapping a project. Wikipedia states that bootstrapping usually refers to the starting of a self-sustaining process that is supposed to proceed without external input.3 This is probably where Twitter's Bootstrap got its name when Twitter had to come up with a solution for internal inconsistencies. When I say Bootstrap, I am referring to the collection of tools created by Twitter engineers for the creation of websites and web applications.

There are people today, such as myself, that like to create websites from scratch. This includes building up the layout, CSS, and scripting components from scratch. If this is you, then you may not be interested in Bootstrap at all. However, I would urge you to continue reading in the hopes I can change your mind. As much fun as it may be to build a site from the ground up, it takes a lot of time spent doing repetitive tasks. Bootstrap allows both advanced and beginning users to build a site that looks professional and is supported on all the major browsers in a relatively short amount of time.

Bootstrap comes complete with some elegant, minimal CSS. In fact, most everything you experience when using Bootstrap is based on the implementation of the CSS. It also has optional JavaScript files, that I highly recommend, that allow for commonly needed, smooth actions such as dropdowns, navigation bars, tabbing, unique buttons, collapsible items, and photo viewing sliders. The time you save in implementing common items is well worth the investment to learn the Bootstrap classification terms. I would encourage everyone to at least make one page with Twitter's Bootstrap before passing judgment, and I will explain how you can do that.

How to Get Going

To get started, you will want to bookmark the Bootstrap Hompage. There you will find all the useful documentation for using Bootstrap. There are plenty of examples to get you going but we will cover a few here as well. You will want to click on the button "Download precompiled Bootstrap" that is highlighted in blue. The current distribution is version 3.0.3 and includes Bootstrap's CSS, both compiled and minified, files for font objects and symbols, and JavaScript files, again compiled and minified variations. Unzip the downloaded folder somewhere your web server can link to the files included. (Note: some features in Bootstrap may not work if you are not viewing pages via a webserver)

The following are all that need to be included in your HTML document to get started.4 (Note: if you choose to use javascript, you will need to link to a jQuery library too)

<!DOCTYPE html>
<html>
  <head>
    <title>Bootstrap 101 Template</title>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Bootstrap -->
    <link href="css/bootstrap.min.css" rel="stylesheet">

    <!-- HTML5 Shim and Respond.js IE8 support of HTML5 elements and media queries -->
    <!-- WARNING: Respond.js doesn't work if you view the page via file:// -->
    <!--[if lt IE 9]>
      <script src="https://oss.maxcdn.com/libs/html5shiv/3.7.0/html5shiv.js"></script>
      <script src="https://oss.maxcdn.com/libs/respond.js/1.3.0/respond.min.js"></script>
    <![endif]-->
  </head>
  <body>
    <h1>Hello, world!</h1>

    <!-- jQuery (necessary for Bootstrap's JavaScript plugins) -->
    <script src="https://code.jquery.com/jquery.js"></script>
    <!-- Include all compiled plugins (below), or include individual files as needed -->
    <script src="js/bootstrap.min.js"></script>
  </body>
</html>

That's it! You are ready to start using Bootstrap. Start exploring the different features of Bootstrap:

  • CSS, Layouts, and Responsive Design
  • Bootstrap Components
  • Bootstrap Specific JavaScript
  • Customize your own Bootstrap
Once you feel comfortable with the classifications that Bootstrap has assigned, you will be able to experiment with customizing your own distributions. Also, do not be afraid to get into the CSS files yourself and make changes. Here is an example of some changes I made to get more options for color coded zebra striping for my work (the original code only included striping for the color gray):

/* since bootstrap (table-striped) uses nth-child(), we just use this to add a zebra stripe color */
.tablesorter-bootstrap tr.odd td {
	background-color: #f9f9f9;
}
.tablesorter-bootstrap tr.even td {
	background-color: #fff;
}

There are many personalized changes you can make. Bootstrap is great for experimenting with new concepts. All the basic styling is done, giving you the freedom to do more advanced styling with your free time. And for those businesses that would just like to get a site up and running to have a web presence, Bootstrap has all the basic eye appeal you would need right out of the box.

Simple Examples

One of the main reasons I like Bootstrap is for the sleek table features. Also, the grid layout is very easy to understand. RWD is another great feature that Bootstrap developers have implemented and continue to improve. Below I will show some snippets from the Bootstrap site on how to put some of these features into your code.

Tables

To get the basic Bootstrap table styling, add the attribute class="table" to the table element. If you like "zebra striping," you can simply add a class .table-striped to the table element. Be aware that the striping feature uses the :nth-child CSS selector which is not compatible with some older browsers. By adding the class table-hover to the table element, your table rows will highlight as you move the mouse over them. For smaller devices such as phones, adding the class .table-responsive to the table element allows users to scroll tables horizontally on screens smaller than 768 pixels. If you implemented all these table features, your code would look something like this: 

<table class="table table-striped table-hover table-responsive">
	...
</table>

Grid Layout

If you are used to Bootstrap 2.x like I am, grids are going to be a little different in 3.x. The old version used div elements with classes like .span1, .span8, etc. In Bootstrap 3.x this had been replaced with classes like .col-xs-1, .col-lg-8, etc. The new version is much more supportive of mobile devices, so in addition to defining the number of columns, it also defines what size of screen the layout applies to with the middle portion of the class name. The options are .col-xs-#, .col-sm-#, .col-md-#, and .col-lg-#. In order to achieve the following layout, you would implement the following code:

  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
  <div class="col-md-1">.col-md-1</div>
</div>
<div class="row">
  <div class="col-md-8">.col-md-8</div>

Responsive Design with Bootstrap

With the plethora of device sizes that can access the web, it is important to either develop a version of your site for each device or to develop in such a way that your site can instantly respond. The latter option is definitely the best. That is what we refer to when we talk about Responsive Web Design (RWD). Bootstrap 2.x gave the option for responsive elements, but with version 3.x, it comes standard. Besides the scalability of the layout, you can add classes to your elements to make them either visible or hidden for different device sizes. The following tables give an idea of how to make elements visible or hidden with this simple implementation of Bootstrap and CSS.

Available Classes to Assist with RWD
Extra Small Devices (<768px) Small Devices (≥768px) Medium Devices (≥992px) Large Devices (≥1200px)
.visible-xs Visible Hidden Hidden Hidden
.visible-sm Hidden Visible Hidden Hidden
.visible-md Hidden Hidden Visible Hidden
.visible-lg Hidden Hidden Hidden Visible
.hidden-xs Hidden Visible Visible Visible
.hidden-sm Visible Hidden Visible Visible
.hidden-md Visible Visible Hidden Visible
.hidden-lg Visible Visible Visible Hidden

Why Upgrade from Bootstrap 2.x to 3.x?

One last note to those that already use a previous version of Bootstrap but may be hesitant to upgrade: If it will cost you enormous amounts of time and money to implement version 3.0.3 without any gain, then perhaps you should not upgrade. If, however, RWD is important to you, you should probably take the time to upgrade. Even though version 2.x includes some aspects of RWD, version 3.x was redesigned with mobile in mind. It will definitely make a difference in your experience of trying to appeal to a wide variety of devices.

Have fun with Twitter's Bootstrap!
